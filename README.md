# SYNTHEX Holding Website

Bilingual static corporate experience for SYNTHEX Holding and its four operating companies:

- JOLLAQ
- Al Maria
- SYNTHEX Industrial
- SHAMCO LLC

The site uses supplied concept renders as responsive interactive scenes with pointer depth, keyboard-accessible hotspots, company-specific themes, synchronized hash navigation, and full Arabic RTL support.

## Requirements

- Node.js 22 or newer
- npm 10 or newer
- Microsoft Edge for browser verification, or an `EDGE_PATH` environment variable pointing to a Chromium executable

## Setup

```powershell
npm install
npm run dev
```

Development routes:

- `http://localhost:3000/en/`
- `http://localhost:3000/ar/`

## Commands

```powershell
npm run lint
npm run typecheck
npm test
npm run build
npm run browser:verify
npm run hydration:verify
npm run check
```

`npm run check` runs linting, TypeScript, unit tests, the production export, browser QA, and hydration QA.

## Production

The Next.js project uses `output: "export"`. A production build is written to `out/`.

```powershell
npm run build
npm start
```

The included static preview server runs at `http://127.0.0.1:3000` by default. Set `PORT` to use another port.

The deployment host must:

- Serve directory routes through `index.html`.
- Preserve `/en/` and `/ar/`.
- Serve `.txt` React Server Component payloads generated by Next.js.
- Serve AVIF and WebP with the correct content types.
- Preserve URL hashes and browser history behavior.

## Architecture

- `app/(localized)/[locale]/`: statically generated English and Arabic routes and metadata.
- `components/experience/ExperiencePage.tsx`: navigation state, section observation, history, language switching, and page composition.
- `components/experience/*World.tsx`: company-specific content and interactive scene states.
- `components/experience/ResponsiveSceneImage.tsx`: responsive image selection, pointer depth, loading feedback, and accessible hotspots.
- `components/experience/HoldingSections.tsx`: holding overview, story, principles, unified network, contact routing, and footer.
- `lib/content.ts`: bilingual company configuration and contextual navigation.
- `lib/experience-state.ts`: hash, section, and active-company reducer.
- `public/media/`: optimized AVIF and WebP derivatives shipped to production.
- `source-assets/`: preserved supplied originals; excluded from production.
- `scripts/verify-browser.mjs`: responsive, interaction, RTL, history, fallback, and reduced-motion verification.
- `scripts/verify-hydration.mjs`: extension-attribute hydration regression verification.

## Content Editing

Company configuration and navigation live in `lib/content.ts`. Holding-level content lives in `components/experience/HoldingSections.tsx`.

Keep English and Arabic content structurally aligned. Do not remove verification notices until the relevant claim is approved.

The following client content is still required:

- Approved vision, mission, and values.
- Registered legal names and registration details.
- Headquarters and operating addresses.
- Approved email, phone, inquiry routing, and social links.
- Privacy and contact-consent language.
- Approved leadership spelling, title, biography, and image permission.
- Evidence and disclosure permission for contract, coverage, timeline, and project claims.
- Production domain for absolute canonical, sitemap, and social-sharing URLs.

## Assets

Responsive scene files follow this convention:

```text
public/media/<company>/<scene>-640.avif
public/media/<company>/<scene>-960.avif
public/media/<company>/<scene>-1122.avif
public/media/<company>/<scene>-640.webp
public/media/<company>/<scene>-960.webp
public/media/<company>/<scene>-1122.webp
```

Preserve the originals in `source-assets/`. Do not replace or redraw official logos without approval.

The current favicon is a provisional geometric holding-system mark and must be replaced when an approved official vector favicon is supplied.

## Accessibility and Motion

- Semantic text and navigation remain outside images.
- Hotspots are keyboard-accessible buttons.
- Company selector navigation supports arrow keys, Home, End, and Escape.
- Focus styles are visible.
- Arabic uses document-level RTL direction.
- `prefers-reduced-motion` disables scene movement, ambient animation, loading sweeps, and smooth scrolling.
- No canvas or WebGL is required, so the full experience remains available without WebGL.

## SEO

Localized titles, descriptions, canonical paths, language alternates, Open Graph metadata, headings, and Organization schema are included.

Absolute canonical URLs, sitemap URLs, and social preview image URLs are intentionally not configured until the production domain is approved.
